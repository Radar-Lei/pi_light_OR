<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI flags -> subprocess runner | User-supplied block names trigger local experiment commands. |
| Existing artifacts -> manifest | JSON/CSV/Markdown files become evidence for reproducibility claims. |
| README/reports -> claim audit | Human-facing text can overclaim beyond Phase 3/4 evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Tampering | `scripts/reproduce_blocks.py --run` | mitigate | Use a fixed internal registry; reject unknown block names; no shell=True. |
| T-05-06 | Repudiation | `reproducibility_manifest.json` | mitigate | Include command strings, expected artifacts, parse status, row/result counts, and generation timestamp if implemented. |
| T-05-07 | Information Disclosure | Manifest paths | accept | Manifest records local repository-relative artifact paths only; no secrets or external service data. |
| T-05-08 | Denial of Service | `--block all` or closed-loop main reruns | mitigate | Default to list/audit; require explicit `--run --block closed_loop_main` for longer SUMO runs. |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install task is planned. |
</threat_model>
