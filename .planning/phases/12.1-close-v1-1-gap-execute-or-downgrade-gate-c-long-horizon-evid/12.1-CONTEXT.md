**Gathered:** 2026-05-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 12.1 closes the remaining v1.1 contradiction around Gate C long-horizon evidence. Phase 11 and Phase 12 correctly built fail-closed machinery and reproducibility packaging, but the current main Gate C source artifact is still `INCONCLUSIVE` because zero of the predeclared long-horizon rows were executed under the runtime guard.

This phase must execute the original predeclared Phase 11 main profile and rerun strict Gate C, then update downstream reproducibility/status artifacts so v1.1 no longer appears complete while its long-horizon dominance evidence is missing. If execution does not produce a complete, paired, valid main-profile dataset, the phase must fail closed and preserve `FAILED`/`INCONCLUSIVE` status rather than converting partial rows into dominance evidence.

This phase must not draft manuscript text, broaden the claim beyond binding regimes, introduce new benchmark families, weaken Gate C requirements, or reinterpret Phase 10 smoke/spec rows as dominance evidence.

</domain>

<decisions>
## Implementation Decisions

### D-12.1-01 Evidence Route
- Execute the full original Phase 11 `main` profile rather than downgrading Gate C without running evidence.
- Preserve the predeclared scope: 6 binding stress scenarios × 6 controllers × 20 paired seeds × 3 demand multipliers = 2160 expected rows.
- The execution target is the existing Phase 11 evidence artifact family, especially `experiments/dual_sensitivity/phase11_long_horizon_paired_seed_evidence.json` and `experiments/dual_sensitivity/phase11_gate_c_paired_evidence.json`.

### D-12.1-02 Fail-Closed Handling
- Any missing row, unpaired seed, missing required comparator, SUMO/runtime failure, missing stress metadata, missing explicit finite-storage fields, or strict Gate C failure must leave the evidence as `FAILED` or `INCONCLUSIVE`.
- Partial completed raw rows may be retained for audit/provenance, but they must not make strict Gate C pass or become binding-regime dominance evidence.
- If full execution finishes but Gate C fails statistically, record the failure honestly; do not tune thresholds or narrow the evidence family after seeing results.

### D-12.1-03 Scope of Remediation
- After execution, rerun strict Gate C and regenerate any Phase 12 reproducibility/claim-status artifacts that consume Phase 11/Gate C status.
- Requirements/status language may be updated only to reflect actual executed evidence: `PASSED`, `FAILED`, or `INCONCLUSIVE` with row counts and reasons.
- Keep all claims simulator-, network-, horizon-, seed-, demand-multiplier-, and binding-regime-relative.

### Claude's Discretion
- Downstream agents may decide safe execution mechanics: batching, resumable row execution, checkpointing, parallelism, logging, and timeout management.
- Downstream agents may add resume/progress support if needed to complete 2160 rows safely, but must not change the predeclared evidence scope unless execution proves the existing runner is technically impossible.

</decisions>
