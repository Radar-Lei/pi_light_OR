# v1.4 Research Summary

## Research Result

v1.4 should proceed as a method-search milestone with strict claim discipline. The target is strong baseline dominance, but the current evidence does not support that claim yet. The correct structure is exploratory parallel search followed by one locked confirmation protocol.

## Key Findings

- v1.3 fully executed the main profile, so the problem is not missing rows.
- The failed primary metric rule contains 48 bounded-harm comparisons, 221 inconclusive comparisons, and 10 non-worsening comparisons.
- The strongest harms appear in delay/travel-time rows for downstream blockage and spillback-style groups, often against all required strong comparators.
- Existing Phase 7/9 static evidence remains useful but cannot carry the closed-loop superiority claim.
- The existing Phase 11/13/12 machinery is valuable and should be reused with v1.4-specific candidate IDs and locked protocols.

## Recommended v1.4 Workstreams

| Workstream | Purpose | Exit Criterion |
|------------|---------|----------------|
| `v1-4-score-controller` | Improve live score/action rule | Candidate improves pilot Gate C without new bounded harm |
| `v1-4-objective-weights` | Tune objective components and thresholds | Frozen weights pass pilot non-worsening checks |
| `v1-4-scenario-diagnostics` | Diagnose whether scenarios activate the intended binding mechanism | Report identifies claim-relevant stress families and non-claimable families |
| `v1-4-symbolic-policy` | Recover/compress a stronger candidate into auditable symbolic rules | Compressed policy preserves pilot advantage and audit fields |

## Roadmap Implication

The milestone should have five main phases:

1. v1.3 failure diagnostics and workstream protocol.
2. Parallel exploratory method workstreams.
3. Workstream convergence and candidate lock.
4. Predeclared v1.4 Gate C execution.
5. Claim/reproducibility refresh and milestone audit.

## Claim Boundary

Closed-loop superiority over max-pressure-style baselines remains disallowed until a v1.4 locked Gate C artifact is `PASSED`. Pilot wins, static separation, and scenario diagnostics are not manuscript-ready dominance evidence.
