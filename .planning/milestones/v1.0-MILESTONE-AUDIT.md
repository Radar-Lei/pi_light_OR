## Tech Debt

### Phase 2 / Phase 3 artifact handoff

Phase 3 consumes Phase 2 recovery implementation directly rather than consuming Phase 2 generated JSON/CSV/rules outputs. This does not break the static kill-gate flow, but it weakens artifact-level handoff semantics.

Affected requirements: RECV-05, KILL-02, KILL-05, REPR-04, REPR-05.

### Closed-loop baseline feasibility

`local_pilight` and `full_dual_symbolic` are marked `not_feasible` in closed-loop artifacts. This is honestly disclosed and not a blocker, but it remains future scientific/engineering debt.

Affected requirement: CLOP-02.

### Root planning-file path expectation

The real planning files are `.planning/ROADMAP.md` and `.planning/REQUIREMENTS.md`; root-level `ROADMAP.md` and `REQUIREMENTS.md` do not exist. This does not break the reproduction flow, but it may surprise fresh readers or external tooling.

Affected requirements: REPR-01, REPR-03.

### Forbidden phrase manifest false positives

Reproducibility and paper artifact manifests store forbidden phrase lists for claim-discipline checks. Naive grep over the artifacts may report false positives even though the actual README, reports, and paper rows do not contain overclaim language.

Affected requirements: REPR-04, REPR-05.
